# Services Overview

## Zweck

Die Services bilden die funktionale Ebene von **1210 MethodOS™**.
Während der Core beschreibt *was* existiert, beschreiben Services *wie*
Wert erzeugt wird.

# Grundprinzipien

Jeder Service besitzt:

-   eindeutigen Zweck
-   definierte Inputs
-   definierte Outputs
-   KPIs
-   Owner
-   unterstützende Capabilities
-   Method Scripts
-   Evidenzbasis

# Governance

## MUSS

-   wiederverwendbar sein
-   dokumentiert sein
-   messbare Ergebnisse liefern

## SOLL

-   unabhängig von einzelnen Personen funktionieren
-   automatisierbar sein

## DARF NICHT

-   Implementierungswissen mit Service-Logik vermischen
