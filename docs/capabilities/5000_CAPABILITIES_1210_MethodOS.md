# Capabilities Overview

## Zweck

Capabilities beschreiben die wiederverwendbaren fachlichen Fähigkeiten
von 1210 MethodOS™. Eine Capability beantwortet nicht, **wer** etwas
tut, sondern **welche Fähigkeit** das System zuverlässig beherrscht.

## Governance

### MUSS

-   messbar sein
-   dokumentiert sein
-   versioniert sein
-   mindestens einer Implementierung zugeordnet sein

### SOLL

-   modular sein
-   mehrfach nutzbar sein

### DARF NICHT

-   an genau ein Produkt gebunden sein

## Ziel

Capabilities bilden den fachlichen Baukasten, aus dem sämtliche
Produkte, Beratungsleistungen, Schulungen und Softwaremodule von 1210
MethodOS™ zusammengesetzt werden.

> Capabilities are the reusable engines of value creation.
