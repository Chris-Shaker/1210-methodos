# Präambel

## Einleitung

1210 MethodOS™ entsteht aus der Überzeugung, dass Hospitality kein
zufälliges Handwerk, sondern ein systematisch entwickelbares Fachgebiet
ist.

Diese Constitution bildet den normativen Kern des Systems. Sie definiert
die Grundannahmen, die Sprache und die Richtung aller zukünftigen
Entwicklungen.

------------------------------------------------------------------------

# Ausgangspunkt

Hospitality ist heute häufig geprägt durch:

-   implizites Wissen
-   personenabhängige Qualität
-   uneinheitliche Standards
-   fehlende Dokumentation
-   geringe Wiederverwendbarkeit

MethodOS verfolgt einen anderen Ansatz.

Wissen wird dokumentiert.

Dokumentation wird strukturiert.

Strukturen werden messbar.

Messungen ermöglichen Verbesserung.

------------------------------------------------------------------------

# Selbstverständnis

MethodOS ist:

-   ein Referenzsystem
-   ein Wissenssystem
-   eine Entwicklungsplattform
-   ein Qualitätsrahmen
-   eine Grundlage für KI-gestützte Systeme

MethodOS ist ausdrücklich keine einzelne Software und kein einzelnes
Unternehmen.

------------------------------------------------------------------------

# Leitbild

Alle Implementierungen -- Department 66, StiegerConsult,
Cocktail-Analyst, CaaS, D66 Academy und zukünftige Plattformen -- dienen
einem gemeinsamen Zweck:

**Die kontinuierliche Weiterentwicklung des Referenzsystems.**

------------------------------------------------------------------------

# Verpflichtung

Jede Erweiterung von MethodOS soll:

1.  nachvollziehbar sein,
2.  dokumentiert sein,
3.  versioniert sein,
4.  evidenzbasiert begründet werden,
5.  die Konsistenz des Gesamtsystems erhalten.

------------------------------------------------------------------------

# Grundsatz

> MethodOS entwickelt nicht nur bessere Hospitality-Unternehmen.
>
> MethodOS entwickelt eine bessere Methode, Hospitality-Unternehmen zu
> entwickeln.
