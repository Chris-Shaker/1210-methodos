# Capability Model

## Zweck

Das Capability Model beschreibt die wiederverwendbaren fachlichen
Fähigkeiten von 1210 MethodOS™. Capabilities sind unabhängig von
einzelnen Unternehmen, Personen oder Softwarelösungen und bilden die
Brücke zwischen Methodik und Implementierung.

------------------------------------------------------------------------

# Definition

Eine **Capability** ist die nachweisbare Fähigkeit eines Systems, einen
definierten Zweck reproduzierbar zu erfüllen.

Eine Capability besteht aus:

-   Zweck
-   Inputs
-   Outputs
-   Regeln
-   KPIs
-   Wissen
-   Verantwortlichkeiten
-   unterstützenden Services

# Regeln

## MUSS

-   klar definierten Nutzen besitzen
-   messbar sein
-   dokumentiert sein
-   versioniert sein

## SOLL

-   mehrfach wiederverwendbar sein
-   unabhängig von einzelnen Personen funktionieren

## DARF NICHT

-   ausschließlich an eine konkrete Implementierung gebunden sein.

Services, SOPs und Referenzimplementierungen zusammengesetzt werden.
