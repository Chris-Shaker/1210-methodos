# Product Model

## Zweck

Das Product Model beschreibt, wie aus den Bausteinen von 1210 MethodOS™
marktfähige Produkte und Dienstleistungen entstehen.

Produkte sind Implementierungen von Capabilities und Services auf Basis
des gemeinsamen MethodOS-Kerns.

------------------------------------------------------------------------

# Produktarchitektur

``` text
Foundation
      │

      │

      │

      │

      │

      │
───────────────
      │

```

# Produktregeln

## MUSS

-   auf MethodOS basieren
-   dokumentiert sein
-   versioniert sein
-   definierte Zielgruppen besitzen

## SOLL

-   Capabilities wiederverwenden
-   wirtschaftlich skalierbar sein
-   messbaren Nutzen liefern

## DARF NICHT

-   den Kern von MethodOS verändern
-   proprietäres Wissen unkontrolliert duplizieren

------------------------------------------------------------------------

# Beziehung zu den Referenzimplementierungen

Die Referenzimplementierungen erzeugen Erkenntnisse.

MethodOS standardisiert diese Erkenntnisse.

Produkte machen die Standards wirtschaftlich nutzbar.

------------------------------------------------------------------------

# Ziel

Ein gemeinsamer Methodenkern soll viele unterschiedliche Produkte
ermöglichen, ohne dass Wissen oder Qualität fragmentiert werden.

> One Core. Many Products.
