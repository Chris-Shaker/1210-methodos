# Information Flow

## Zweck

Dieses Dokument definiert den Fluss von Daten, Informationen, Wissen und
Evidenz innerhalb von 1210 MethodOS™.

## Informationslebenszyklus

``` text
Beobachtung
    │
    ▼
Daten
    │
    ▼
Information
    │
    ▼
Wissen
    │
    ▼
Evidenz
    │
    ▼
Standard
    │
    ▼
Anwendung
    │
    ▼
Feedback
```

## Informationsquellen

-   Referenzimplementierungen
-   Forschung
-   Audits
-   KPIs
-   Gästefeedback
-   Teamfeedback
-   Software
-   Academy

## Informationssenken

-   Standards
-   SOPs
-   Frameworks
-   Trainings
-   AI-Agenten
-   Produkte

## Flussregeln

### MUSS

-   Informationen erhalten eine Quelle.
-   Wissen wird versioniert.
-   Änderungen werden dokumentiert.

### SOLL

-   Doppelte Informationen vermeiden.
-   Wissen zentral pflegen.
-   Evidenz mit Quellen verknüpfen.

### DARF NICHT

-   Operatives Wissen ausschließlich personenbezogen gespeichert werden.
-   Widersprüchliche Referenzstände parallel gepflegt werden.

## Rolle der Referenzimplementierungen

Department 66 und weitere Implementierungen erzeugen kontinuierlich neue
Informationen. Diese werden geprüft, bewertet und in das zentrale
Wissenssystem übernommen.

## Ziel

Ein geschlossener Informationskreislauf, der Lernen, Standardisierung
und kontinuierliche Verbesserung ermöglicht.

> Daten werden zu Wissen. Wissen wird zu Standards. Standards erzeugen
> bessere Entscheidungen.
