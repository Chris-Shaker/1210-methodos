# Application Model

## Zweck

Das Application Model beschreibt, wie 1210 MethodOS™ in
unterschiedlichen Organisationen, Produkten und Geschäftsbereichen
angewendet wird.

Es definiert die Beziehung zwischen dem gemeinsamen Kernsystem und
seinen Referenzimplementierungen.

------------------------------------------------------------------------

# Architektur

``` text
                 1210 MethodOS™

                       │
        ┌──────────────┼──────────────┐
        │              │              │
   Department 66   StiegerConsult   D66 Academy
        │              │              │
        ├──────────────┼──────────────┤
        │              │
 Cocktail-Analyst      CaaS
        │
 Forensischer Barkeeper™
```

------------------------------------------------------------------------

# Referenzimplementierungen

## Department 66

Living Research Laboratory.

Funktion:

-   Entwicklung
-   Validierung
-   Testbetrieb
-   KPI-Messung

------------------------------------------------------------------------

## StiegerConsult

Transformation Platform.

Funktion:

-   Beratung
-   Strategie
-   Implementierung
-   Audits

------------------------------------------------------------------------

## Cocktail-Analyst

Hospitality Intelligence Platform.

Funktion:

-   Analyse
-   Daten
-   KI
-   Wissenszugriff

------------------------------------------------------------------------

## CaaS

Continuous Capability as a Service.

Funktion:

-   kontinuierliche Organisationsentwicklung
-   Capability-Aufbau
-   Performance-Steigerung

------------------------------------------------------------------------

## D66 Academy

Capability Development Platform.

Funktion:

-   Ausbildung
-   Zertifizierung
-   Trainings
-   Wissensvermittlung

------------------------------------------------------------------------

## Forensischer Barkeeper™

Diagnostic Platform.

Funktion:

-   Ursachenanalyse
-   Qualitätsprüfung
-   Auditmethodik

------------------------------------------------------------------------

# Gemeinsame Grundlage

Alle Implementierungen verwenden dieselben:

-   Axiome
-   Gesetze
-   Prinzipien
-   Standards
-   Objekte
-   Capabilities
-   Services

------------------------------------------------------------------------

# Ziel

Neue Unternehmen oder Produkte sollen MethodOS übernehmen können, ohne
den Kern des Systems verändern zu müssen.

> One Method. Multiple Implementations.
